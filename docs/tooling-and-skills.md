# 插件、软件和 Skills 推荐清单

这份清单按你当前目标整理：做 AI 产品作品集、面试展示、AI 视频工作流、数据分析项目，以及自定义 Agent/Skill。

## 已安装且建议继续用

- Codex GitHub 插件：适合看 PR、Issue、创建 PR、修 CI、处理 review。
- Codex Browser 插件：适合验证本地网页、截图、跑前端体验检查。
- Codex Chrome 插件：适合操作你已经登录的网站，比如 ChatGPT、TapNow、GitHub 网页。
- Codex Documents / Spreadsheets / Presentations：适合做简历、项目文档、Excel、PPT。
- `ai-framework-interview-coach` skill：适合把你的项目包装成面试回答。
- `hatch-pet` skill：适合展示你会做 Codex 自定义资产和工具链。
- `jianying-workflow` skill：适合剪映/视频制作流程。
- `pdf`、`playwright`、`screenshot-intake`：适合分析 PDF、网页自动化、截图排查。

## 建议安装的 Codex Skills

- `openai-docs`：查 OpenAI API/模型/Agents SDK 官方文档时用。
- `skill-creator`：把你的工作流沉淀成自己的 skill，适合面试展示。
- `plugin-creator`：如果你要做 Codex 插件作品集，用它脚手架化。
- `github:yeet`：把本地改动整理成分支、commit、PR。
- `github:gh-fix-ci`：修 GitHub Actions 失败。
- `github:gh-address-comments`：处理 PR review 评论。
- `vercel-deploy`：前端 demo 一键部署到 Vercel。
- `screenshot` 或 `screenshot-intake`：继续用于视觉审核。
- `transcribe` / `speech`：如果你做视频字幕、口播、配音流程，会很有用。

## 建议安装的软件

- GitHub CLI：命令是 `gh`，你当前电脑里还没有识别到。它能登录 GitHub、创建仓库、推送、开 PR。
- Git for Windows：Git 基础工具，建议确认已安装并在 PATH。
- Node.js LTS：你现在 Node 已有，但 AI/前端项目建议用 LTS 版本更稳。
- VS Code：管理项目、README、Git 变更更方便。
- Power BI Desktop：你已经在用，适合继续做数据分析作品。
- OBS Studio：录制作品集 demo 视频。
- ShareX：快速截图、标注、GIF。
- FFmpeg：视频抽帧、转码、合成必备。
- Jianying / CapCut：你的视频作品最终剪辑工具。

## GitHub 作品集项目建议

- 一个总览仓库：`ai-portfolio-hub`。
- 一个前端/AI 产品 demo：展示页面、交互、产品逻辑。
- 一个数据分析项目：Power BI + 数据生成脚本 + 业务分析说明。
- 一个 AI 视频 workflow 项目：分镜、提示词、关键帧、成片链接。
- 一个 Agent/Skill 项目：展示你能把流程产品化、自动化。

## 面试展示优先级

1. 先把 `README` 做好：面试官最先看这个。
2. 每个项目放 1-3 张高质量截图。
3. 每个项目写清楚输入、流程、输出、你的判断。
4. 能在线打开的 demo 优先级最高。
5. 不要把所有过程废图都上传，仓库要像作品集，不像临时文件夹。
